# E-Market Multiplataforma - Organización del Proyecto

## Estructura de carpetas

- `/src`: Organización del código fuente (clases/archivos vacíos según arquitectura)
- `/adrs`: Decisiones de arquitectura (Architecture Decision Records)
- `/docs`: Diagramas y documentación complementaria (C4, clases, métricas)
- `/presentacion`: Material para la defensa oral

## ¿Cómo está organizado el código?

El código está organizado siguiendo los principios definidos en los ADRs y la arquitectura seleccionada. Cada carpeta representa un módulo o componente crítico del sistema, facilitando la extensibilidad y el mantenimiento.

## ¿Qué contiene cada carpeta?

- **src/**: Estructura de paquetes, módulos o capas según el estilo arquitectónico elegido.
- **adrs/**: Documentos ADR numerados y con índice.
- **docs/**: Diagramas C4, diagrama de clases, cálculo de métricas.
- **presentacion/**: Recursos para la exposición oral.
